======= GIT COMMIT AND PUSH =======

### A. COMMIT AND PUSH NORMAL

1) Daftarkan file mana saja yang ingin Anda kirim.

    git add .

        # (Tanda titik `.` artinya: "Daftarkan semua perubahan yang tidak dilarang oleh .gitignore")

2) Commit (Penguncian Versi)
    # Bungkus perubahan tersebut dengan label pesan.

        git commit -m "Notes"

3) Push (Pengiriman ke Cloud)**
    # Kirim bungkusan tadi ke server GitHub.

        git push origin main


### B. Penanganan Kondisi Khusus (Troubleshooting)

Jika sempat mengalami kendala "rejected" dan file bocor:

1) A. Jika Muncul Error "Rejected" (Gara-gara edit di Web)
    # Jika sempat menghapus file manual di website GitHub, harus menarik data terbaru dulu agar sinkron:

        git pull origin main --rebase

        git push origin main

2) Jika Ingin Memaksa (Force Push)
    # Gunakan ini hanya jika yakin data di komputer adalah yang paling benar dan ingin menghapus sampah yang ada di GitHub:

        git push origin main --force

### C. Cara Cek Status (Navigasi Radar)
    # Sebelum melakukan commit, sangat disarankan untuk selalu mengecek kondisi "Radar" Git Anda agar tidak ada file rahasia yang bocor lagi:

1)  Cek daftar file yang berubah:
        
        git status
    
        # (Pastikan `.env` atau file `.xlsx` tidak muncul di daftar hijau/merah ini)

2)  Cek riwayat pengiriman terakhir:
    
    git log --oneline -n 5
